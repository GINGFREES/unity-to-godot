DOTween（Unity）vs Tween（Godot）

官方資料來源

  • DOTween 文件：https://dotween.demigiant.com/documentation.php
  • Godot Tween 類別：https://docs.godotengine.org/zh-tw/4.x/classes/class_tween.html

Unity：DOTween

  • 用途：一套第三方（免費 + 付費「Pro」等級）的 tween 引擎，用來以直覺、可鏈式呼叫的 API 為數值／物件製作動畫。
  • 核心概念：個別的 Tweeners 為單一數值製作動畫；Sequences 則將多個 tween／回呼組合成一條時間軸，並支援巢狀結構。
  • 建立方式：可使用 Unity 型別上的捷徑方法（transform.DOMoveX()），或針對任意數值使用泛用的 getter/setter lambda tween。
  • 鏈式呼叫慣例：DO* 建立一個 tween、Set* 設定它、On* 附加回呼——例如 transform.DOMoveX(4, 1).SetLoops(2, LoopType.Yoyo)。
  • 數值型別：float、double、int、向量（vectors）、四元數（quaternions）、顏色、字串，並可透過外掛支援自訂型別。
  • 迴圈：Restart、Yoyo（每個循環先反向播放再正向播放）、以及 Incremental（數值隨每個循環持續累加）。
  • 緩動（Ease）：單一統一的 Ease 列舉（例如 OutQuad、InOutQuint），或自訂 AnimationCurve／函式。
  • 回呼：OnStart、OnComplete、OnKill、OnPlay、OnPause、OnRewind，直接附加在 tween／sequence 上。
  • 生命週期安全性：可選用的「Safe Mode」會自動處理目標物件被銷毀時的 tween。
  • DOTween Pro（付費）：新增像是螺旋 tween 及 2D Toolkit 整合等額外功能。

Godot：Tween

  • 用途：內建、輕量、以腳本驅動的動畫系統，用來隨時間對數值做插值——不需要第三方素材。相較於 AnimationPlayer，官方將其定位為更高效能的替代方案，適合用在簡單／動態的動畫上（例如事先未知的數值）。
  • 建立方式：呼叫 some_node.create_tween()，接著串接 tween_property()（為物件屬性製作動畫）、tween_method()（每一步以插值後的數值呼叫函式）、tween_callback()（在序列的某一步觸發任意函式）、tween_interval()（一個計時延遲步驟）。
  • 排序（Sequencing）：各步驟預設依序執行；.parallel() 會讓某個步驟與前一步同時執行；.chain() 則在 parallel 群組之後明確回到依序執行。
  • Ease／Transition：結合兩個獨立的列舉——TransitionType（LINEAR、SINE、ELASTIC、BOUNCE、QUAD……）與 EaseType（IN、OUT、IN_OUT、OUT_IN）——而非 DOTween 那種單一統一的 Ease 列舉。
  • 迴圈：set_loops(count)（不帶參數表示無限次）。
  • 控制：play()／pause()、stop()（重置但不移除 tweeners）、kill()（中止並使其失效）。
  • 重要注意事項（來自官方文件）：
    • Tween 並非設計成可重複使用——每個動畫都應建立新的 Tween。
    • Tween 建立後會立即開始——只有在動畫真正應該開始時才建立它。
    • 避免讓兩個 tween 同時對同一個屬性做動畫——最後建立的那一個會生效。
    • 將 tween 綁定到某個節點（生命週期管理），以避免它比其動畫對象存活更久。

回答可能會被問到的問題

1. Godot 是否有內建的 DOTween 對應方案？
有——Tween 是引擎原生功能（不需要素材／外掛），且涵蓋相同的核心範疇：屬性 tweening、排序、緩動／過渡曲線、迴圈以及回呼。

2. DOTween Pro 的額外功能是否有涵蓋到？
沒有直接涵蓋。螺旋 tween、punch/shake 捷徑，以及 2D Toolkit 整合都是 DOTween Pro 特有的功能；如果專案有使用這些功能，就需要在 Godot 中手動實作（通常是透過 tween_method() 驅動一個自訂的插值函式）。

3. 哪些 API 差異對遷移來說真正重要？

  • 緩動統一化：建立一份對照表，將專案中使用到的每個 DOTween Ease 值對應到一組 Godot 的 TransitionType+EaseType。
  • 迴圈／Yoyo：確認 set_loops() 是否原生支援乒乓效果，還是只會重複正向播放（見後續追蹤事項）——可能需要為每個循環手動撰寫一個正向步驟加上一個反向步驟。
  • 生命週期回呼：DOTween 那六個細緻的回呼並不會與 Godot 的三個信號一對一對應；有些（例如序列中途的 OnComplete）用 tween_callback() 步驟來表達會比用 Tween 層級的信號更合適。
  • 物件銷毀的安全性：將對 DOTween Safe Mode 的依賴替換為明確呼叫 bind_node()。

4. 整體遷移的大致樣貌
大多數 DOTween 呼叫在概念上都能一對一轉換——捷徑方法或泛用 tween 會變成 create_tween().tween_property(...)，而 DOTween 的 Sequence 會變成單一 Godot Tween 的預設依序鏈結，並在原本 DOTween 使用 .Join() 的地方插入 .parallel()。

後續追蹤事項／待日後確認

  • 確認 Godot 的 set_loops() 是否原生支援 Yoyo／乒乓迴圈，還是只會正向重複——目前擷取到的文件片段尚未確立此點，在假設其與 LoopType.Yoyo 等效之前需先驗證。
  • 盤點專案實際上對哪些 DOTween 數值型別／外掛做 tween（標準的 float／向量／顏色，還是自訂外掛型別）——Godot 以屬性為基礎的 tweening 可直接涵蓋 Godot 原生型別；自訂型別可能需要使用 tween_method()。
  • 確認專案中是否真的有使用到 DOTween Pro 的功能（螺旋 tween、punch/shake、2D Toolkit）。
  • 檢查 Unity 程式碼中是否有逐畫格／高頻率建立 tween 的模式——Godot 明確不鼓勵隨意重複使用／重新建立 tween，因此在 DOTween 中運作良好的模式，在 Godot「每個動畫建立新 Tween」的模型下可能需要重新思考。
